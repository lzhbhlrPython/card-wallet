## [1.4.0] - 2026-01-13
### Added
- 证件管理功能：支持护照、身份证（ID卡）、出入境通行证、驾驶证等证件类型。
  - 新增 `documents` 表，存储证件信息（持有人姓名、证件号码、有效期、签发地点、备注）。
  - 证件信息使用 RSA-2048 加密，私钥经 AES-256-CBC 加密后存入数据库。
  - 新用户注册时自动生成 RSA 密钥对；旧用户登录时自动补充密钥对。
  - 列表接口 `/documents` 返回掩码后的证件号码（显示首尾各2字符，中间用星号替代）、持有人姓名和证件类型，无需 2FA。
  - 详情接口 `/documents/:id` 需要 2FA 验证，返回完整证件信息（持有人姓名、拉丁转写、证件号码、有效期、签发地点、备注）。
  - 支持多语种姓名：持有人本国语言名称 + 可选的拉丁字母转写（用于国际证件）。
  - 有效期支持多种显示格式：年/月/日 (YMD)、月/日/年 (MDY)、日/月/年 (DMY)。
- **日历组件 (Calendar.vue)**：封装统一的日期选择器，支持自定义日期格式和"长期"选项，应用于证件管理。
  - 支持快速选择年份和月份的下拉选择器。
  - 月份显示使用英文缩写（Jan, Feb, Mar, Apr, May, Jun, Jul, Aug, Sep, Oct, Nov, Dec）。
- **证件有效期增强**：
  - 新增签发日期字段 (`issue_date`)，支持记录证件签发时间。
  - 支持长期有效证件标识 (`expiry_date_permanent`)，勾选后无需填写具体有效期。
  - 证件表单使用日历组件替代原有文本输入，提升用户体验。
- **Modal全局提示组件**：
  - 新增 `Modal.vue` 和 `useModal.js` composable，提供统一的信息提示、确认对话框功能。
  - 支持 info/success/warning/error 四种类型，可自定义标题、按钮文本。
  - 已集成到 App.vue，可在整个应用中使用（后续将替换系统 alert）。
- 前端证件管理页面：
  - `/documents` 列表页面：展示证件类型、持有人、掩码证件号、备注；支持按类型筛选和排序。
  - `/documents/new` 创建页面：填写证件信息（类型、姓名、拉丁转写、号码、有效期、格式、签发地点、备注）。
  - `/documents/:id/edit` 编辑页面：加载和更新证件信息，需要 2FA。
  - 列表页面支持查看详情（2FA 弹窗）、编辑、删除操作。
- 前端已实现 token invalid 自动登出功能（401/403 invalid token 时自动清除状态并跳转登录页）。

### Changed
- 数据库 `users` 表新增 `rsa_public_key` 和 `encrypted_rsa_private_key` 字段。
- 数据库 `documents` 表新增 `encrypted_issue_date` 和 `expiry_date_permanent` 字段。
- 包含向后兼容的自动迁移逻辑（`ALTER TABLE`）。
- 导航栏新增"证件"链接，位于"转数快"和"备份"之间。
- 版本号更新为 1.4.0。
- **证件详情页面优化**：
  - 模态框采用渐变背景 + 白色浮出层设计，提高可读性（与卡片详情页风格一致）。
  - 按正确顺序展示证件信息：类型 → 姓名 → 拉丁转写 → 证件号码 → 签发日期 → 有效期至 → 签发地点 → 备注。
  - 支持复制签发日期和有效期。
  - 长期证件显示"长期"标识而非具体日期。

- **密码安全增强**：
  - 注册、登录、清空数据时，客户端使用 MD5 对密码进行哈希后再传输至后端，避免明文密码暴露。
  - 后端存储 bcrypt(MD5(password))，实现双重哈希保护。
- **证件号码掩码统一**：掩码后的证件号码字符串统一为 8 位（格式：XX****XX）。

### Fixed
- 证件详情模态框样式与卡片详情保持一致，解决可读性问题。
- 日期格式化逻辑统一，支持 YMD/MDY/DMY 三种格式的正确显示。

### Security
- 证件信息使用更强的 RSA 非对称加密（2048位），私钥用 AES 加密存储。
- 除掩码证件号、持有人姓名、证件类型外，所有证件信息（包括完整证件号、有效期、签发地点、拉丁转写）均需 2FA 验证后才能查看。
- 签发日期使用 RSA 加密存储，保持与其他敏感字段相同的安全级别。
- 密码传输安全：客户端发送 MD5(password)，服务器存储 bcrypt(MD5(password))，网络拦截和数据库泄露均无法直接获取原始密码。

## [1.0.0] - 2025-08-12
### Added
- 初始发布：Vue 3 + Vite 前端（登录、注册、2FA 设置、卡片列表、创建、详情、编辑、删除、备份、退出登录）。
- Express + SQLite 后端：用户注册/登录、JWT 鉴权、TOTP 双因素认证、卡片 CRUD、字段 AES-256-CBC 对称加密（独立 IV）。
- 卡组织自动识别：Visa / Mastercard (51–55, 2221–2720) / American Express / Discover / JCB / Maestro / UnionPay / MIR / Diners / T-Union / eCNY / Unknown。
- Luhn 校验（对特殊网络的豁免处理逻辑预留）。
- WebDAV 手动备份能力（前端表单 + 时间戳文件名）。
- 2FA 设置、验证、重置相关端点与页面流程。
- Python 测试脚本基础版本（批量生成多网络卡片并调用后端 API）。
- 发布包含 `.gitignore`（忽略数据库与敏感文件）与 GPLv3 LICENSE。 

### Changed
- README 标题与克隆命令统一为 “Card Wallet”。

## [1.1.0] - 2025-08-12
### Added
- 后端新增 `POST /cards/purge` 端点：需要主密码 + 2FA（若启用）以清空当前用户全部卡片。
- 前端导航栏新增红色“清空所有卡片”按钮，自定义模态输入主密码与 2FA 验证码。
- 测试脚本 `server_test.py` 增加 `--rounds` 参数支持多轮批量创建。

### Changed
- 测试脚本修正 Mastercard 2-series 生成逻辑，使用官方合法 BIN 区间 222100–272099，并保证 6 位 BIN。
- 后端 Mastercard 识别逻辑由正则改为数值范围判断，避免漏判 2-series 边界值。
- 测试脚本 eCNY CVV 固定为 000。
- README 文档更新 UI 与 Mastercard 说明。

### Security
- 清空端点要求双重验证，降低误操作或滥用风险。

## [1.2.0] - 2025-08-13

- FPS (转数快) 账户管理：列表 / 创建 / 详情 (2FA) / 编辑 / 删除，全量后端 REST 接口与前端页面。

- 修复 `/fps/banks` 被 `/fps/:id` 捕获的路由顺序问题。

- 银行 Logo 命名统一为：原名大写→非字母数字替下划线→转小写；前端按规则生成。
- Purge 前端文案改为“清空信息”，提示包含 FPS。

- 增加前端 invalid token 处理：403 + "Invalid token" 或 401 自动登出跳转登录页并携带标记参数。

### Security

### Notes

## [1.3.0] - 2026-01-12
### Added
- 后端新增 `cards.card_type` 列（枚举字符串），并在 API 层对其进行验证与网络特例强制处理（`tunion` → `transit`，`ecny` → `ecny_wallet_1..4`）。
- 前端：在卡片创建/编辑表单增加类型选择（中文显示，提交 English enum），`BankSelect` 增加 `readonly`/`allowCreate`/`allowClear` 支持以防止非法输入。
- 卡片详情保留为列表内弹窗查看（需 2FA）；独立路由 `/cards/:id` 已移除以避免重复入口点。
- 测试脚本 `server_test.py` 更新：创建测试卡时包含 `cardType` 并兼容特殊网络规则。

### Changed
- 卡片列表增加“类型”筛选与“按类型”排序，排序使用固定顺序：信用卡 / 借记卡 / 预付卡 / 公交卡 / 一类钱包 / 二类钱包 / 三类钱包 / 四类钱包。
- `CardItem` 调整展示布局：`类型` 在左侧，`有效期` 在右侧并列，字体权重与颜色统一以便快速识别。

### Notes
- 数据库兼容性：本次发布引入 `cards.card_type` 字段。服务端会在启动时尝试自动迁移以添加新列，但请务必先备份 `server/data/database.sqlite`。旧客户端（< v1.3.0）可能无法正确处理 `cardType` 字段；请同时升级前端和后端到 v1.3.0 以确保一致性。
